Reference

SnapAPI HTTP API

所有业务接口均为 POST， Content-Type: application/json。 生产环境基址示例：https://snapapi.up.railway.app

鉴权 · Authentication

调用 /screenshot 与 /pdf 时，必须在请求头携带 x-api-key，值为你在数据库中为用户配置的 API Key（与 Dashboard 中展示的一致）。

请求头

x-api-key: <YOUR_API_KEY>

请勿将 Key 提交到公开仓库；生产环境建议使用环境变量或密钥管理服务注入。

端点 · Endpoints

POST /screenshot

使用无头浏览器打开目标 URL，截取页面为 PNG 并上传至对象存储，响应体返回可访问的图片 URL。

  • 默认整页截图（fullPage 默认为真）；可改为仅视口。
  • 可选 width / height 成对指定视口，影响布局与截图区域。
POST /pdf

同样基于 Chromium 渲染页面并导出 PDF，上传后返回下载链接。fullPage 对 PDF 无单独语义；页面尺寸由视口与内容决定。

  • 若同时提供 width 与 height，将用该视口渲染，并以相同尺寸作为 PDF 页面大小。
  • 若省略二者，则按文档滚动宽高自动计算 PDF 页面尺寸。

成功响应 JSON

{
  "status": "success",
  "message": "Screenshot saved" | "PDF saved",
  "path": "https://..."
}

请求体参数 · Parameters

JSON 对象字段说明如下。

字段 类型 默认值 说明
url string —（必填） 要加载的 HTTP(S) 页面地址。两个端点均必填。
fullPage boolean true 仅影响 /screenshot：true 截取完整可滚动页面； false 仅当前视口。PDF 端点忽略此字段。
width integer Playwright 默认视口 视口宽度，范围 320–4096。必须与 height 同时传入才会生效；仅传其一将被忽略。
height integer Playwright 默认视口 视口高度，范围 240–4096。PDF 模式下与 width 一起决定页面尺寸。

错误码 · Error codes

错误时响应体一般为 JSON：{ "error": "..." }。

400 Bad Request

请求体不符合 JSON Schema（例如缺少 url、类型错误、数值超出允许范围）。

401 Unauthorized

未携带 x-api-key，或 Key 在数据库中不存在 / 不匹配。

429 Too Many Requests

触发限流或配额：全局限流（见下文）或当月 API 调用次数超过套餐 max_limit（提示信息类似 Quota exceeded for current period）。

500 Internal Server Error

服务端未捕获异常、渲染或上传失败等。可结合服务日志与 requestId（若反向代理注入）排查。

另：数据库未配置时，截图/PDF 可能返回 503（Service Unavailable）。

限制 · Limits

  • 全局限流：单 IP 约 5 次请求 / 1 minute（/health 不计入）。超出返回 429。
  • 月度配额：按用户 usage_count 与 max_limit 统计；超额返回 429。
  • 页面加载超时：默认 60s（环境变量 SCREENSHOT_GOTO_TIMEOUT_MS 可调整）。超时可能导致 500。

SDK 示例

以下为最小可运行示例，请将基址与 API Key 换为你的环境值。

Node.js（axios）

const axios = require("axios");

const base = "https://snapapi.up.railway.app";
const apiKey = process.env.SNAPAPI_KEY;

async function screenshot() {
  const { data } = await axios.post(
    `${base}/screenshot`,
    { url: "https://example.com", fullPage: true },
    { headers: { "Content-Type": "application/json", "x-api-key": apiKey } }
  );
  console.log(data.path);
}

screenshot().catch(console.error);

Python（requests）

import os
import requests

BASE = "https://snapapi.up.railway.app"
API_KEY = os.environ["SNAPAPI_KEY"]

def export_pdf():
    r = requests.post(
        f"{BASE}/pdf",
        json={"url": "https://example.com", "width": 1280, "height": 720},
        headers={"x-api-key": API_KEY},
        timeout=120,
    )
    r.raise_for_status()
    print(r.json()["path"])

if __name__ == "__main__":
    export_pdf()
